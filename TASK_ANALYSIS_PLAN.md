# تحليل وتقرير تطوير نظام طمطوم للتوصيل

## القسم الأول: تطبيق السائق (Driver App)

### 1.1 وضع عدم الاتصال (Offline Mode)
**الحالة الحالية:**
- لا يوجد دعم للعمل بدون internet
- جميع الطلبات تتم عبرfetch API مباشرة للخادم
- عند انقطاع الإنترنت، يفشل التطبيق بالكامل

**المطلوب:**
- إضافة Service Worker للتخزين المؤقت
- حفظ الطلبات المحلية في IndexedDB
- مزامنة البيانات عند استعادة الاتصال
- عرض حالة "غير متصل" للسائق

### 1.2 نظام التواصل (Chat)
**الحالة الحالية:**
- يوجد جدول `messages` في قاعدة البيانات
- يوجد مكون `DriverCommunication.tsx` لكن غير واضح مدى التكامل
- لا يوجد نظام دردشة حقيقي مع العميل أو المطعم

**المطلوب:**
- ربط نظام الرسائل بالكامل
- إضافة دردشة بين السائق والعميل
- إضافة دردشة بين السائق والمطعم
- إشعارات فورية للرسائل الجديدة

## القسم الثاني: تطبيق العميل (Client App)

### 2.1 حسابات العملاء
**الحالة الحالية:**
- ✅ صفحة تسجيل الدخول/التسجيل موجودة (CustomerAuthPage.tsx)
- ✅ جدول المستخدمين موجود في قاعدة البيانات
- ❌ لا يوجد حفظ للعناوين بشكل كامل
- ❌ لا يوجد سجل طلبات محفوظ للعميل

**المطلوب:**
- إكمال نظام حفظ العناوين (userAddresses)
- عرض سجل طلبات العميل السابق
- حفظ المفضلة

### 2.2 بوابة الدفع
**الحالة الحالية:**
- ✅ واجهة اختيار الدفع موجودة (نقدي، بطاقة، محفظة، دفع إلكتروني)
- ❌ خيار "البطاقة" هو نص فقط بدون ربط فعلي
- ❌ لا يوجد تكامل مع بوابات دفع حقيقية

**المطلوب:**
- إنشاء جدول إدارة طرق الدفع
- ربط مع بوابات دفع (Stripe, PayPal, أو بوابات محلية)
- إضافة صفحة إعدادات الدفع في لوحة التحكم

## القسم الثالث: لوحة التحكم (Admin Dashboard)

### 3.1 التقارير المتقدمة والرسوم البيانية
**الحالة الحالية:**
- ✅ يوجد صفحة AdvancedReports.tsx
- ❌ لا توجد رسوم بيانية (Charts)
- ❌ تقارير السائقين والمطاعم موجودة لكن بدون رسوم بيانية

**المطلوب:**
- إضافة مخططات بيانية (Charts) باستخدام مكتبة Chart.js أو Recharts
- تقارير شهرية وسنوية
- تحليل الأداء

### 3.2 سجل العمليات (Audit Log)
**الحالة الحالية:**
- ✅ يوجد مكون AuditLogViewer.tsx
- ✅ يستدعي /api/admin/audit-logs
- ❌ غير مرتبط بصفحات الإدارة بالكامل

**المطلوب:**
- ربط جميع عمليات المسؤولين بالسجل
- تسجيل من قام بتعديل الطلب أو الحالة

### 3.3 تقارير جديدة
**المطلوب إضافة:**
- تقارير السائقين (أرباح، طلبات، تقييمات)
- تقارير العملاء (عدد الطلبات، قيمة المشتريات)
- تقارير المنتجات (الأكثر مبيعاً، التقييمات)
- تقارير المطاعم (المبيعات، العمولات)

## القسم الرابع: قاعدة البيانات والسيرفر

### 4.1 الجداول المطلوبة
```sql
-- ✅ messages (الدردشة) - موجود
-- ✅ auditLogs (سجل العمليات) - موجود
-- ✅ paymentGateways (بوابات الدفع) - موجود
-- ❌ 需要 تكامل الكامل
```

### 4.2 مسارات API المطلوبة
- `/api/messages` - للرسائل
- `/api/admin/audit-logs` - موجود لكن يحتاج تحسين
- `/api/admin/payment-gateways` - طرق الدفع
- `/api/client/addresses` - عناوين العميل
- `/api/client/orders/history` - سجل الطلبات
